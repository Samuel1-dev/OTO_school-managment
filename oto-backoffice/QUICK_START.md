# Guide Rapide - OTO School Back Office\n\n## Installation et Démarrage\n\n### Étape 1: Installer les dépendances\n\n```bash\nnpm install\n```\n\n### Étape 2: Lancer le serveur de développement\n\n```bash\nnpm start\n```\n\nL'application sera accessible à `http://localhost:4200`\n\n### Étape 3: Configuration du Backend\n\nAssurez-vous que le backend API est en cours d'exécution sur `http://localhost:3000`\n\n## Flux d'Utilisation\n\n### 1. Connexion\n\n**URL:** `http://localhost:4200/backoffice/login`\n\n**Identifiants de test:**\n- Email: `admin@backoffice.com`\n- Mot de passe: `motdepasse123`\n\n**Étapes:**\n1. Entrez votre email\n2. Entrez votre mot de passe\n3. Cliquez sur \"Se connecter\"\n\n**Après la première connexion:**\nSi le système indique que vous devez changer votre mot de passe, vous serez redirigé vers la page de changement de mot de passe.\n\n### 2. Tableau de Bord\n\n**URL:** `http://localhost:4200/backoffice/dashboard`\n\n**Sections principales:**\n\n#### Statistiques\n- **Total établissements** - Nombre total d'écoles inscrites\n- **En attente** - Écoles en attente de validation (avec badge rouge si > 0)\n- **Validées** - Écoles approuvées\n- **Rejetées** - Écoles rejetées\n\n#### Onglets de Filtrage\n- **En attente** - Affiche uniquement les écoles en attente (onglet par défaut)\n- **Toutes** - Affiche toutes les écoles\n- **Validées** - Affiche les écoles approuvées\n- **Rejetées** - Affiche les écoles rejetées\n\n#### Recherche\n- Tapez pour rechercher par:\n  - Nom de l'école\n  - Ville\n  - Email\n\n#### Tableau des Écoles\n\nPour chaque école, vous pouvez:\n\n**Voir les détails:**\n1. Cliquez sur l'icône \"Oeil\" dans la colonne Actions\n2. Un modal s'ouvre avec les informations complètes\n3. Cliquez sur \"Valider l'établissement\" pour approuver\n4. Cliquez en dehors du modal pour fermer\n\n**Valider une école:**\n1. Cliquez sur l'icône \"Checkmark\" dans la colonne Actions\n2. Confirmez votre décision\n3. L'école passe au statut \"Validée\"\n\n**Rejeter une école:**\n1. Cliquez sur l'icône \"Croix\" dans la colonne Actions\n2. Entrez la raison du rejet\n3. Cliquez sur \"Confirmer le rejet\"\n4. L'école passe au statut \"Rejetée\"\n\n### 3. Déconnexion\n\nCliquez sur le bouton \"Déconnexion\" dans la sidebar pour:\n- Supprimer le token JWT\n- Retourner à la page de connexion\n- Nettoyer la session\n\n## Structure de l'Authentification\n\n### Token JWT\n\n1. **Obtention du token:**\n   - Au moment de la connexion via `POST /backoffice/login`\n   - Réponse: `{ access_token: \"...\", role: \"...\", mot_de_passe_change: true/false }`\n\n2. **Stockage:**\n   - Token stocké dans `localStorage` sous la clé `access_token`\n   - Rôle stocké dans `localStorage` sous la clé `role`\n\n3. **Utilisation:**\n   - Automatiquement ajouté à toutes les requêtes `/backoffice/`\n   - Format: `Authorization: Bearer <token>`\n\n4. **Expiration:**\n   - Si le token expire, l'interceptor détecte l'erreur 401\n   - Redirection automatique vers la page de connexion\n   - Token et rôle sont effacés\n\n## Gestion des Erreurs\n\n### Erreurs Courantes\n\n**\"Identifiants incorrects\"**\n- Vérifiez votre email et mot de passe\n- L'email doit être exact (case-sensitive pour certaines parties)\n\n**\"Impossible de charger les écoles\"**\n- Vérifiez que le backend API est en cours d'exécution\n- Vérifiez la connexion réseau\n- Vérifiez que vous êtes toujours authentifié\n\n**\"Erreur lors de la validation\"**\n- Vous n'avez peut-être pas les permissions nécessaires\n- L'école peut avoir été supprimée par un autre admin\n- Essayez de rafraîchir la page\n\n## Fonctionnalités Avancées\n\n### Filtrage Avancé\n\n1. Utilisez les onglets pour voir les écoles par statut\n2. La barre de recherche filtre instantanément\n3. Combinaison: Vous pouvez chercher \"Paris\" dans l'onglet \"En attente\"\n\n### Notifications\n\n- **Toast vert** - Action réussie (validation, rejet)\n- **Toast rouge** - Erreur ou avertissement\n- Les notifications disparaissent automatiquement après 3 secondes\n\n### Formatage des Dates\n\nLes dates sont affichées au format français:\n- `31 mai 2026 à 14:30`\n\n## Troubleshooting\n\n### Page blanche après connexion\n1. Vérifiez la console (F12 > Console)\n2. Vérifiez que le backend API répond\n3. Vérifiez que le `access_token` est présent dans `localStorage`\n4. Essayez `Ctrl+Shift+R` pour un cache refresh complet\n\n### Logout automatique\n- Vérifiez que votre jeton JWT n'a pas expiré\n- Reconnectez-vous\n- Augmentez la durée du JWT sur le backend si nécessaire\n\n### Erreur CORS\n1. Vérifiez que le backend a CORS activé\n2. Vérifiez l'URL du backend dans `auth.service.ts`\n3. Assurez-vous que la réponse a les bons headers:\n   ```\n   Access-Control-Allow-Origin: http://localhost:4200\n   Access-Control-Allow-Methods: GET, POST, PATCH, DELETE\n   Access-Control-Allow-Headers: Content-Type, Authorization\n   ```\n\n## Development\n\n### Tester avec des données fictives\n\n1. Modifiez le backend pour retourner des données de test\n2. Ou modifiez `dashboard.component.ts` pour ajouter des données en dur:\n\n```typescript\nngOnInit(): void {\n  // Données de test\n  this.ecoles = [\n    {\n      id: '1',\n      nom: 'École Test',\n      ville: 'Paris',\n      email: 'test@ecole.com',\n      telephone: '+33123456789',\n      boite_postale: '75000',\n      nom_fondateur: 'John Doe',\n      status: 'en_attente'\n    }\n  ];\n  this.chargerEcolesPending();\n}\n```\n\n### Déboguer les requêtes API\n\n1. Ouvrez les Developer Tools (F12)\n2. Allez à l'onglet \"Network\"\n3. Effectuez une action (connexion, validation, etc.)\n4. Cliquez sur la requête pour voir:\n   - Headers\n   - Request body\n   - Response body\n   - Status code\n\n### Inspecter le stockage local\n\n1. F12 > Application\n2. Local Storage > http://localhost:4200\n3. Cherchez `access_token` et `role`\n\n## Variables d'Environnement (Future)\n\nCréez un fichier `environment.ts` pour les configurations:\n\n```typescript\nexport const environment = {\n  production: false,\n  apiUrl: 'http://localhost:3000'\n};\n```\n\nPuis mettez à jour `auth.service.ts`:\n\n```typescript\nimport { environment } from '../../../environments/environment';\n\nprivate apiUrl = `${environment.apiUrl}/backoffice`;\n```\n\n## Support et Contact\n\nPour toute question ou problème:\n1. Consultez la console du navigateur pour les erreurs\n2. Vérifiez les logs du backend\n3. Consultez la documentation dans `ARCHITECTURE.md` et `IMPLEMENTATION.md`\n
